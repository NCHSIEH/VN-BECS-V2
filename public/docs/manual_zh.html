VN-BECS V1.0 已全面進化，不再採用線性的 SOP 模式，而是轉化為具備「戰略縱深」的 **6大核心功能區 (Nodes)** 架構。本手冊旨在引導各級人員在對應的功能節點中執行臨床作戰任務，確保從「捐贈者血管」到「病患血管」的全生命週期 (Vein-to-Vein) 絕對安全。

01 捐血中心站 (LIMS)

LIMS 節點負責血液供應鏈的初始採集與身分定義，是數據真實性的唯一源頭。該子系統透過與越南國家身分證資料庫及 ISBT-128 條碼協定標準的數位整合，將捐血者的實體身分與採集血品進行「數位雙胞胎」綁定。

一、 階段式四重作業流 (Four-Stage Workflow)

LIMS 子系統內部運作劃分為四個「戰略階段」，採取單向硬鎖定流程，確保上一階段未完成之血袋無法進入後續加工或調度：

1
Registry Control (登記控制)

收集捐血者基本資料，進行越南 12 位身分證編碼格式校驗與捐血適齡性（16 至 65 歲）即時篩查。

2
Clinical Screening (臨床檢驗篩選)

執行傳染病標記 (IDM) 及核酸檢測 (NAT)。檢驗通過前，系統將血袋狀態永久鎖定為「隔離 (PENDING)」，阻斷 any 物流出庫。

3
Phlebotomy Ops (採血作業)

將核定通過之捐贈案綁定 ISBT-128 DIN（捐贈識別碼）。進行成分血（紅血球、血漿、血小板）的分離製作。

4
Chain of Custody (保管與釋放)

完成所有品質認證與出庫審核，數位發布至調度系統，移交 HUB 供應鏈中心實行調配。

三、 資料欄位與欄位值說明 (Data Fields & Value Definition)

1. 捐贈者登記表單 (Donor Registry Form)

欄位名稱 (Field Name) 資料類型 格式條件 (Validation/Format) 說明與欄位值定義 (Description & Values)
Donor ID (捐贈者代號) String (UUID) 系統唯讀自動產生 捐血者於系統內部的永久識別碼，用於關聯歷次採血紀錄。
Full Name (姓名) String 長度 ≥ 2 字元，英文大寫 捐血者之法定姓名。例：NGUYEN VAN A。不允許純數字。
National ID (CCCD) String 12 位純數字 越南國民身分證編號。首 3 位為省份代碼（001 至 096），第 4 位代表世紀與性別（0-9）。
Date of Birth (出生日期) Date YYYY-MM-DD 捐血者生日。系統將依此即時計算年齡，未成年或超齡者將被拒絕登記。
Blood Type (血型) Enum O | A | B | AB 捐血者的主要 ABO 血型系統類別。
Rh Factor (Rh因子) Enum Positive (+) | Negative (-) RhD 血型系統分類。Negative 屬於罕見血型，會觸發系統高亮黃色標記。

2. 採血初始化表單 (Phlebotomy Initiation Form)

欄位名稱 (Field Name) 資料類型 格式條件 (Validation/Format) 說明與欄位值定義 (Description & Values)
Donation ID (DIN) String 開頭為 =，長度 ≥ 13 字元 ISBT-128 標準捐贈識別碼。格式：=W[機構4碼] [年份2碼] [流水號6碼]。例：=W0000 24 102938。
Target Volume (採血量) Integer 200 至 600 (單位: mL) 本次預計採血的毫升數。預設值為 500。
Collection Type (採血方式) Enum WholeBlood | Apheresis WholeBlood: 全血採集（血量 200~600mL）
Apheresis: 分離術採集（血量上限為 300mL，超額會觸發警報）

四、 安全聯鎖「硬閘門」與防錯條件 (Safety Interlocks & Hard Gating Rules)

為落實臨床 100% 安全防護，系統於前端與後端資料庫層面均部署了「強制連鎖閘門 (Hard Gating)」，當條件不滿足時，操作按鈕會呈現灰色鎖定，並伴隨醒目的紅色 SAFETY INTERLOCK ACTIVE 警示：

SYSTEM SECURITY GATE POLICY
  • 身份安全閘門 (Identity Gate): 身分證字號必須通過 LocalCCCDValidator 算法校驗。若長度不足 12 位、首 3 位省份碼不符合越南官方對應範圍 (001-096)，登記按鈕強制鎖定，並顯示錯誤提示。
  • 年齡限制閘門 (Age Verification Gate): 依據 Date of Birth 即時計算的年齡必須落在開設定位區間內（越南標準為 16 至 65 歲）。若計算年齡 < 16 或 > 65 歲，系統直接拒絕送出，從源頭杜絕超齡或未成年採血之法規風險。
  • 試劑到期閘門 (Expired Reagent Lock): 若庫存主數據管理模組 (MDM) 中，當前檢測用試劑 (Reagent) 的狀態被標記為 Expired，LIMS 實驗室檢驗模組的 Process Diagnostics 按鈕立即失能，系統拋出試劑過期硬阻斷。
  • 儀器維保閘門 (Analyzer Maintenance Lock): 若檢測分析儀 (Analyzer) 的設備狀態為 MaintenanceRequired，系統封鎖全部檢測運行操作，提示維保警報。
  • 設備維護閘門 (Centrifuge Maintenance Lock): 若血液離心機 (Centrifuge) 設備狀態為 MaintenanceRequired，系統阻斷 Fabricate Components 製作流程，強制要求完成儀器維修後方可開鎖。
  • 陽性隔離閘門 (Bio-risk Quarantine Lock): 若 Donation 案之檢測結果 idmStatus === 'REACTIVE'，Fabricate Components 按鈕永久失效，顯示 QUARANTINE RESTRICTED。系統將該血袋強行鎖定於隔離資料表，禁止任何出庫操作。

02 品質檢驗實驗室 (LAB)

LAB 節點是血液放行的技術守門員，確保每一單位血品皆符合國家安全檢驗標準。

CRITICAL: 隔離機制 (QUARANTINE)

所有新進血品在完成 IDM (傳染病標記) 與 NAT 核酸檢測前，均會被系統鎖定在「強制隔離」狀態，嚴禁調度出庫。

關鍵流程包含自動化 IDM 篩檢、精準 ABO/RhD 定型、以及成分血分離製作 (紅血球、血漿、血小板)。

03 供應鏈調度中心 (HUB)

HUB 節點是系統的物流引擎，負責全國庫存的動態調配與精準派發。

技術核心 作業內容 戰略價值
FEFO 演算法 優先推薦最接近過期日的血袋。 將全國血液浪費率降低至 2% 以下。
IoT 冷鏈追蹤 運送箱溫度每 5 分鐘實時上傳。 確保血液在運輸過程中活性不受損。

04 臨床醫療節點 (HOSPITAL)

HOSPITAL 節點是臨床執行端，負責病患端的訂血、配血與最後一道輸血安全防線。

1
B2B 數位訂血

護理人員透過系統直接與 HUB 對接，分為常規 (Routine)、急件 (ASAP) 與極急件 (STAT) 三種等級。

2
床邊掃描雙重驗證

在輸血前，必須同時掃描病患手環與血袋條碼。若血型不合或條碼錯誤，系統會觸發「強制鎖定」，防止意外發生。

05 戰略指揮中心 (NATIONAL)

NATIONAL 節點為國家級指揮官提供宏觀視野，監控全國血液供應的安全與韌性。

功能包含實時全國庫存地圖、物流效率矩陣，以及針對突發大量傷亡事件的「危機模擬推演」工具。

06 主數據管理中心 (MDM)

MDM 是系統的行政大腦，管理所有組織節點、帳號權限與產品目錄。

ADMIN GOVERNANCE

系統管理員必須在此節點配置 RBAC 角色政策，確保不同職能的人員僅能訪問對應的功能區塊。
