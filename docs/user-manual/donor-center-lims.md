## 1. 子系統概述

捐血中心 LIMS（Laboratory Information Management System）模擬 eProgesa 平台，負責管理捐血者從報到到血液成分發送至 Hub 的完整臨床流程。

本系統遵循：
- **VN26**（越南衛生部血液安全規範）
- **AABB** 技術手冊（捐血間隔、採血量標準）

### 工作流程架構

```
 ┌────────────────┐     ┌─────────────┐     ┌─────────────────┐     ┌──────────────────┐
 │  DONOR（登記）  │────▶│  LAB（檢測） │────▶│  PROCESS（製備） │────▶│  RELEASE（發血）  │
 └────────────────┘     └─────────────┘     └─────────────────┘     └──────────────────┘
  捐血者登記               IDM 血清學            成分離心製備              Hub 監管鏈移交
  健康問卷篩檢             CLEARED/REACTIVE      RBC / FFP / PLT          稽核記錄
  靜脈穿刺排隊派送                               triage 座椅管理
```

| 段落 | 頁籤 | 功能 |
|------|------|------|
| **DONOR** | 登記管理 | 捐血者登記、健康問卷篩檢、靜脈穿刺排隊派送 |
| **LAB** | 臨床診斷 | IDM 血清學檢測（CLEARED / REACTIVE / PENDING） |
| **PROCESS** | 成分製備 | 離心製備 RBC / FFP / PLT，triage 座椅管理 |
| **RELEASE** | 監管鏈 | 血液成分發送至 Hub，監管鏈交接驗證 |

## 3. 操作流程

### 3.0 整體流程總覽

一名捐血者完整通過系統的路徑如下：

```
捐血者到場
    │
    ▼
[DONOR 頁籤] 登記表單 ──── CCCD / 年齡 / 體重驗證 ────▶ 驗證失敗？──▶ 顯示錯誤，禁止送出
    │
    ▼ 驗證通過
健康問卷填寫
    │
    ├── 有暫緩項目？──▶ 是 ──▶ 問卷送出（deferralStatus = Active）
    │                              ├─▶ 問卷強制儲存（失敗則整筆中止）
    │                              └─▶ Toast「捐血者已登記（暫緩）」→ 禁止採血 → 結束
    │
    └── 無暫緩 ──▶ 問卷送出（deferralStatus = Clear）
                     ├─▶ 問卷強制儲存
                     ├─▶ 捐血者記錄建立
                     └─▶ 自動跳出派送對話框

[派送對話框] 選擇模式（Auto / Shared / Direct）
    │
    ▼
[PROCESS 頁籤] triage 椅子分配
    │
    ▼
採血完成（靜脈穿刺記錄）
    │
    ▼
[LAB 頁籤] IDM 血清學送檢
    │
    ├── 試劑過期？──▶ 是 ──▶ 紅色安全閘門，操作封鎖
    ├── 設備需維護？──▶ 是 ──▶ 紅色安全閘門，操作封鎖
    │
    └── 安全 ──▶ 送檢 ──▶ 結果回寫
                              ├─▶ CLEARED ──▶ 可進行成分製備
                              └─▶ REACTIVE ──▶ 標示「QUARANTINE RESTRICTED」，封鎖製備

[PROCESS 頁籤] 成分製備（IDM = CLEARED）
    │
    └─▶ 製備 RBC / FFP / PLT
    │
    ▼
[RELEASE 頁籤] Hub 發血移交
    │
    ├── 成分已過期？──▶ 是 ──▶ 400 TRANSITION_BLOCKED
    │
    └── 未過期 ──▶ 狀態更新 HUB IN-TRANSIT，寫入稽核記錄
```

### 流程 1：捐血者登記（DONOR 頁籤）

#### 前置條件
- 已使用具備 `LIMS_Simulator` 或 `DonorScreener` 角色的帳號登入
- 系統顯示 **DONOR** 頁籤

#### 步驟說明

```
步驟 1  點選右上角 [Register Donor（＋）] 按鈕
        → 登記表單從右側滑入（全屏蓋板模式）
        → 標題顯示「Donor Enrollment」

步驟 2  輸入 CCCD 身分證號（12 碼）
        → 系統即時驗證省碼（VietnamIdValidator）
        → 若此 CCCD 已有捐血紀錄：自動載入歷史資料
          (顯示 ✓ toast：「Existing donor record auto-loaded」)

步驟 3  系統同時檢查捐血間隔（AABB 12 週規則）
        → 間隔不足：表單顯示紅色警示，禁止送出

步驟 4  填寫基本資料：
        ┌─────────────┬──────────────────────────────┐
        │ 欄位         │ 即時驗證行為                  │
        ├─────────────┼──────────────────────────────┤
        │ 姓名         │ 越南姓名格式；禁止數字/特殊符號  │
        │ 出生日期      │ 格式 YYYY/MM/DD；18–65 歲限制  │
        │ 性別         │ 影響懷孕問卷是否顯示            │
        │ 體重（KG）   │ ≥50kg（男）/ ≥45kg（女）       │
        │ 血型 / RhD  │ 下拉選單，無驗證               │
        └─────────────┴──────────────────────────────┘

步驟 5  填寫健康問卷（7 項勾選框）
        → 任一勾選 → 右側即時顯示「暫緩預覽」紅色警示框
        → 顯示暫緩原因 + 暫緩期限

步驟 6  採血資訊（採血量、方式）
        → 採血量依體重自動計算上限（VN26）
        → 超出上限：「Initialize Record」按鈕灰化禁用

步驟 7  點選 [Initialize Record（建立紀錄）] 或 [Save Changes（儲存修改）]
```

#### 決策樹：送出後系統行為

```
送出表單
    │
    ▼
後端驗證（CCCD / 年齡 / 姓名）
    │
    ├── 驗證失敗？──▶ 表單內紅色錯誤訊息，操作中止
    │
    └── 驗證通過
            │
            ▼
        問卷強制儲存（questionnaires 資料表）
            │
            ├── 儲存失敗？──▶ 500 LIMS_DONOR_CREATE_FAILED
            │                   UI 顯示紅色錯誤訊息
            │                   捐血者記錄不建立（原子操作）
            │
            └── 儲存成功
                    │
                    ▼
                捐血者記錄建立（donors 資料表）
                    │
                    ├── deferralStatus = Active？
                    │       └──▶ Toast 顯示「捐血者已登記（暫緩）」
                    │            採血按鈕灰化禁用
                    │            流程結束
                    │
                    └── deferralStatus = Clear？
                            └──▶ 自動開啟派送對話框（見流程 2）
```

#### 注意事項
- 問卷儲存失敗 → 整筆登記中止（SOP 強制：問卷為必要安全記錄）
- 系統不允許未填問卷即建立捐血者記錄

### 流程 2：派送至靜脈穿刺椅

#### 前置條件
- 捐血者成功登記（deferralStatus = Clear）
- 派送對話框自動跳出，或從捐血者列表手動觸發

#### 派送模式說明

```
┌────────────────────────────────────────────────────────────────────┐
│                         派送對話框                                   │
│                                                                      │
│  ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐   │
│  │  🔄 Auto          │  │  👥 Shared        │  │  🎯 Direct       │   │
│  │  自動負載均衡      │  │  共用等候池       │  │  直接指定座椅    │   │
│  │                  │  │                  │  │                  │   │
│  │ 系統自動選擇       │  │ 捐血者加入共用    │  │ 必須選擇目標椅   │   │
│  │ 最短佇列椅子       │  │ 等候池，等護士    │  │ 子編號（必填）   │   │
│  │                  │  │ 手動呼叫         │  │                  │   │
│  └──────────────────┘  └──────────────────┘  └──────────────────┘   │
│                                                                      │
│  選擇後 → 點選 [Dispatch] → Toast 確認訊息                           │
└────────────────────────────────────────────────────────────────────┘
```

#### 各模式詳解

| 模式 | 觸發條件 | 系統行為 | 適用場景 |
|------|----------|----------|----------|
| **Auto** | 選取並 Dispatch | 系統計算所有椅子目前佇列長度，自動分配最短佇列 | 尖峰時段，平均分流 |
| **Shared** | 選取並 Dispatch | 捐血者加入共用等候池（Shared Pool） | 椅子不確定，等護士呼叫 |
| **Direct** | 選取目標椅子並 Dispatch | 捐血者直接綁定到指定椅子 | 特殊需求（殘障、VIP） |

#### PROCESS 頁籤 — 椅子操作

```
PROCESS 頁籤
    │
    ├── 已有捐血者的椅子：顯示捐血者資訊
    │
    └── 空椅子（等候池有人）：顯示 [呼叫下一位] 按鈕
            │
            └──▶ 點選 → 從 Shared Pool 取下一位捐血者
                        等候池為空？→ 顯示紅色提示訊息
```

### 流程 3：IDM 血清學檢測（LAB 頁籤）

#### 前置條件
- 捐血者已完成靜脈穿刺採血
- IDM 狀態 = **PENDING**

#### 步驟說明

```
步驟 1  點選頁籤 [LAB]
        → 顯示所有捐血記錄，狀態欄顯示 PENDING / CLEARED / REACTIVE

步驟 2  找到目標捐血記錄（狀態 PENDING）
        → 點選 [Process Diagnostics（送檢）]

步驟 3  系統執行安全閘門檢查（順序）：

    ╔══════════════════════════════════════════════╗
    ║   安全閘門檢查序列                            ║
    ╠══════════════════════════════════════════════╣
    ║ Gate A：試劑狀態                              ║
    ║   → 試劑 = Expired？                          ║
    ║       是 → 紅色警示：「SAFETY INTERLOCK:      ║
    ║             Reagent [名稱] is EXPIRED」        ║
    ║             操作完全封鎖，需點 [確認知悉] 關閉  ║
    ║       否 → 進入 Gate B                        ║
    ╠══════════════════════════════════════════════╣
    ║ Gate B：分析儀維護狀態                         ║
    ║   → 設備 = MaintenanceRequired？               ║
    ║       是 → 紅色警示：「SAFETY INTERLOCK:       ║
    ║             Equipment [名稱] requires          ║
    ║             maintenance」                      ║
    ║             操作完全封鎖，需點 [確認知悉] 關閉  ║
    ║       否 → 送出樣本至參考實驗室                ║
    ╚══════════════════════════════════════════════╝

步驟 4  檢測完成 → IDM 狀態回寫：

        PENDING
            │
            ├──▶ CLEARED（無傳染病標記）
            │         → PROCESS 頁籤：顯示 [Fabricate Components] 按鈕（啟用）
            │
            └──▶ REACTIVE（陽性）
                      → PROCESS 頁籤：標示「QUARANTINE RESTRICTED」（紅色）
                        Fabricate Components 按鈕永久禁用
                        此捐血者血液進入隔離處理流程
```

#### 安全閘門觸發畫面

```
╔═══════════════════════════════════════════════════════╗
║  ⚠ SAFETY GATE VIOLATION                              ║
║─────────────────────────────────────────────────────  ║
║  SAFETY INTERLOCK: Reagent [X] is EXPIRED.            ║
║  Laboratory testing blocked.                           ║
║                                                        ║
║                         [ Acknowledge（確認知悉）]     ║
╚═══════════════════════════════════════════════════════╝
```

> **注意：** 安全閘門出現時，**必須** 由操作人員手動點選 [確認知悉] 才能關閉，系統不會自動解除。

### 流程 4：成分製備（PROCESS 頁籤）

#### 前置條件
- 採血記錄存在
- IDM 狀態 = **CLEARED**（未通過則封鎖）

#### 步驟說明

```
步驟 1  點選頁籤 [PROCESS]
        → 捲動至「成分製備」表格區塊

步驟 2  檢視安全欄位：
        ┌─────────────────────────────────────────┐
        │  IDM 狀態   │  顯示文字          │  按鈕  │
        ├─────────────┼──────────────────┼────────┤
        │  CLEARED    │  SAFE（綠色）      │  ✅ 啟用 │
        │  REACTIVE   │  QUARANTINE       │  🚫 禁用 │
        │             │  RESTRICTED（紅色） │        │
        │  PENDING    │  BIO-RISK（黃色）  │  🚫 禁用 │
        └─────────────┴──────────────────┴────────┘

步驟 3  點選 [Fabricate Components（製備成分）]

步驟 4  系統後端再次檢查 IDM 狀態（雙層驗證）
        → IDM ≠ CLEARED？→ 403 IDM_NOT_CLEARED

步驟 5  製備完成 → 成分建立（依捐血類型）：
        ┌─────────────────────────────────────────────┐
        │ 捐血類型      │ 製備成分                      │
        ├─────────────┼─────────────────────────────  │
        │ WholeBlood  │ RBC（紅血球）                   │
        │             │ FFP（新鮮冷凍血漿）              │
        │             │ PLT（血小板）                    │
        │ Apheresis   │ 依採集設定（待查 VN26 條款）     │
        └─────────────┴─────────────────────────────  │

步驟 6  成功後系統提示：前往 RELEASE 頁籤進行發血
```

### 流程 5：Hub 發血移交（RELEASE 頁籤）

#### 前置條件
- 血液成分已製備完成
- 成分狀態 = **READY**
- 成分未過期

#### 步驟說明

```
步驟 1  點選頁籤 [RELEASE]
        → 顯示所有成分，狀態欄：READY / HUB IN-TRANSIT / EXPIRED

步驟 2  找到狀態 = READY 的成分
        → 確認 Product Class（RBC / FFP / PLT）
        → 確認 Custody Status

步驟 3  點選 [RELEASE TO HUB（發送至 Hub）]

步驟 4  後端驗證序列：

    系統檢查成分到期日
        │
        ├── 已過期？──▶ 400 TRANSITION_BLOCKED
        │                 UI 顯示紅色錯誤訊息「Release failed: ...」
        │
        └── 未過期
                │
                ▼
            狀態更新：READY → HUB IN-TRANSIT
                │
                ▼
            寫入稽核記錄（Audit Ledger）
                │
                ▼
            Toast 成功訊息（綠色）
            Hub 端可接收此成分
```

#### 狀態轉換圖

```
            製備完成
               │
               ▼
           ┌───────┐
           │ READY │
           └───────┘
               │ 點選 RELEASE TO HUB
               │
               ▼
    ┌─────────────────────┐
    │   後端到期日檢查     │
    └─────────────────────┘
         │           │
         ▼           ▼
    未過期          已過期
         │           │
         ▼           ▼
  ┌──────────────┐  ┌──────────────┐
  │ HUB IN-TRANSIT│  │   BLOCKED    │
  │  （移交成功） │  │  （400 錯誤） │
  └──────────────┘  └──────────────┘
```

### 流程 6：捐血者資料編輯

#### 前置條件
- 捐血者記錄已存在
- 操作角色：Admin / LIMS_Simulator / DonorScreener

#### 步驟說明

```
步驟 1  在 DONOR 頁籤捐血者列表找到目標記錄
        → 點選 [編輯] 圖示

步驟 2  表單以「Modify Registry」模式開啟
        → 自動載入現有資料

步驟 3  修改欄位（驗證規則與登記相同）

步驟 4  健康問卷重新填寫
        → PUT 路由會附加新問卷記錄（保留稽核歷史，不覆蓋舊記錄）

步驟 5  點選 [Save Changes]

步驟 6  後端行為：
        捐血者資料更新（donors 資料表）
            │
            └──▶ 附加新問卷記錄（questionnaires 資料表）
                    │
                    ├── 成功 → { success: true, id }
                    │
                    └── 失敗 → { success: true, id, warning: "questionnaire_save_failed" }
                                捐血者資料已更新，問卷儲存失敗需手動補記
```

> **稽核設計說明：** 問卷記錄採「附加」策略，每次編輯產生新一筆記錄，保留完整決策歷史。這符合 AABB 和 VN26 的稽核軌跡要求。

## 4. 介面欄位規格

### 4.1 捐血者登記表單

| 畫面/區塊 | 欄位名稱 | 欄位型態 | 必填 | 預設值 | 驗證規則 | 備註 |
| :--- | :--- | :--- | :- | :-- | :--- | :--- |
| 捐血者登記 | 姓名 | 文字 | ✅ | — | 越南姓名格式；不得含數字或特殊符號 | 自動轉大寫 |
| 捐血者登記 | CCCD 身分證號 | 數字（12碼） | ✅ | — | 越南 CCCD 省碼驗證；需為 12 位數 | 可掃描晶片卡或手動輸入 |
| 捐血者登記 | 出生日期 | 文字（YYYY/MM/DD） | ✅ | — | 年齡須介於 18–65 歲（VN26） | 自動格式化分隔符 |
| 捐血者登記 | 性別 | 下拉選單 | ✅ | Male | Male / Female | 影響懷孕問卷顯示 |
| 捐血者登記 | 體重（KG） | 數字 | ✅ | — | 男性 ≥50 kg；女性 ≥45 kg | 影響採血量上限 |
| 捐血者登記 | 血型 | 下拉選單 | ✅ | O | O / A / B / AB | — |

## 5. 臨床驗證規則

| 規則 | 標準 | 系統行為 |
|------|------|----------|
| CCCD 格式 | 12 碼，省碼有效（越南身分證標準） | 即時顯示錯誤，禁止送出 |
| 捐血者年齡 | 18–65 歲（VN26） | 即時顯示錯誤，禁止送出 |
| 捐血間隔 | ≥12 週（AABB） | 在 CCCD 輸入時即時檢查；送出時再次確認 |
| 採血量（WholeBlood） | 依體重分級（見 §4.2）（VN26） | 即時顯示錯誤，按鈕禁用 |
| IDM 封鎖 | 成分製備前 IDM 必須 = CLEARED | 前端禁用按鈕；後端回傳 403 IDM_NOT_CLEARED |
| 過期封鎖 | 已過期成分不得發血至 Hub | 後端回傳 400 TRANSITION_BLOCKED |
| 試劑過期閘門 | 試劑狀態 = Expired 時禁止 IDM 送檢 | 顯示紅色安全閘門警示 |
| 設備維護閘門 | 分析儀/離心機 = MaintenanceRequired 時禁止操作 | 顯示紅色安全閘門警示 |

## 6. 系統訊息與錯誤處理

| Hub 發血成功 | 綠色 Toast | 狀態更新為 HUB IN-TRANSIT |
| API 錯誤（網路中斷） | 紅色訊息框 | 顯示 "Network error" |
| 等候池為空時呼叫下一位 | 紅色訊息框 | 顯示等候池為空提示 |
